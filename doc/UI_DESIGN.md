# 《扩张前线》界面设计文档
## UI/UX Design Document

**版本：** 1.0  
**日期：** 2024-11-30  
**基于：** 游戏设计文档 v1.0

---

## 一、文件结构

### 技术实现
```
index.html        # 主HTML文件(DOM结构)
game.js          # 游戏逻辑代码
style.css        # 样式表
```

### 界面切换方式
- 单页应用(SPA)设计
- 所有界面在同一HTML中,通过 `display: none/block` 切换
- 底部固定Tab栏切换主要界面
- 无页面刷新,保持游戏状态

---

## 二、底部Tab导航栏

### Tab列表(固定在底部)
```
[区域] [建造] [科技] [仓库] [地图]
```

### Tab功能
- **区域** - 主游戏界面,显示当前区域详情
- **建造** - 建造新建筑
- **科技** - 科技树研究
- **仓库** - 仓库管理(设置上限、删除资源)
- **地图** - 区域总览与战斗入口

**功能:** 查看当前区域的所有信息和建筑

**布局:**
```

========================================

【当前区域信息】
区域名称: 废弃矿场
槽位: 8/12 已使用

【资源库存】(实时更新)
铁板: 1250/2000  |  铜板: 800/2000
钢板: 150/500    |  煤: 450/1000
电力: 850/1000 ⚡

【昼夜状态】
⏰ 白天 (剩余 2:15)

【建筑列表】
┌─────────────────────────────────────┐
│ 矿机 Mk2 #1                          │
│ 配方: 铁矿 | 状态: 正常 | 速度: 10/秒  │
│ [操作 ▼]                             │
│   → 暂停                             │
│   → 升级到 Mk3 (需要: 铁板×10...)    │
│   → 拆除                             │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│ 组装机 Mk1 #2                        │
│ 配方: 电路板 | 状态: 缺原料(铜板)    │
│ 速度: 0/秒 (暂停中)                  │
│ [操作 ▼]                             │
│   → 切换配方                         │
│   → 启动                             │
│   → 升级到 Mk2                       │
│   → 拆除                             │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│ 燃煤发电站 #3                        │
│ 配方: 煤 → 电力 | 状态: 正常         │
│ 速度: 20电/秒 (消耗煤 1/秒)          │
│ [操作 ▼]                             │
│   → 暂停                             │
│   → 拆除                             │
└─────────────────────────────────────┘

... (所有建筑)

========================================
底部Tab: [区域*] [建造] [科技] [仓库] [地图]
========================================
```

**关键元素:**
- **区域信息:** 区域名、槽位使用情况、资源点
- **资源库存:** 主要资源实时显示(被动刷新,有变动就更新)
- **昼夜状态:** 文字显示当前时间和剩余时间
- **建筑列表:** 
  - 每个建筑显示: 名称、当前配方、状态、生产速度
  - 可展开操作菜单
  - 操作包括: 切换配方、暂停/启动、升级、拆除

**建筑状态类型:**
- 正常
- 缺电
- 缺原料(显示缺什么)
- 仓库已满(无法输出)
- 无配方(暂停)

**操作行为:**
- **切换配方:** 弹出配方选择列表,立即切换,材料返还
- **暂停/启动:** 立即生效
- **升级:** 
  - 检查是否有高级版本(需科技解锁)
  - 显示升级成本(补差价)
  - 确认后立即升级,无等待时间
- **拆除:** 5秒后完成,返还100%资源

```
========================================
底部Tab: [区域] [建造] [科技] [仓库*] [地图]
========================================
```

**关键元素:**
- **容量总览:** 总容量、已用、剩余
- **资源列表:** 显示所有持有的物品
- **每个资源显示:**
  - 物品名称
  - 当前数量
  - 上限设置(下拉菜单或输入框)
  - 占用容量(显示占用规则,如无人机、弹药的特殊占用)

**上限设置:**
- **MAX** - 默认值,表示无上限(有空间就存)
- **自定义数字** - 玩家手动设置上限
- 设置低于当前数量时:
  - 保留当前数量
  - 停止生产该资源
  - 消耗到上限以下后恢复生产

**删除资源功能:**
```
点击[删除...] → 弹出输入框
┌─────────────────────────┐
│ 删除 铁板                │
│ 当前数量: 1000           │
│ 删除数量: [____] (输入框) │
│ [确认] [取消]            │
└─────────────────────────┘
```
- 输入要删除的数量
- 确认后立即删除,释放对应容量
- 用于腾空间

**批量操作:**
- [全部设为MAX] - 将所有资源上限恢复为无限制

### 3.6 战斗准备界面

**功能:** 组建无人机编队,检查资源,发起进攻

**布局:**
```
========================================
       准备进攻区域 8
========================================

【敌军情报】
区域: 腐蚀沼泽
敌军: 中型战虫 × 10, 装甲甲虫 × 5
推荐: 重机枪无人机 + 激光无人机

【我方编队】(20/30 - 当前/上限)
机枪无人机 × 5      [+ 增加] [- 减少]
重机枪无人机 × 10   [+ 增加] [- 减少]
激光无人机 × 5      [+ 增加] [- 减少]

库存:
- 机枪无人机: 15 可用
- 重机枪无人机: 20 可用
- 激光无人机: 8 可用

【资源检查】
✓ 普通子弹: 500/100 (充足)
✓ 穿甲弹: 300/200 (充足)
✓ 电力: 4500/1750 (充足)

【当前时间】
⏰ 白天 (剩余 1:45)
⚠️ 预计战斗时长: 4-5 回合 (约 2 分钟)
⚠️ 战斗可能跨越昼夜!
   - 夜晚太阳能将失效 (-125 电力/秒)
   - 建议充满蓄电池或等待下一个白天

【战斗预测】
预计回合数: 4-5 回合
胜率: 85%

```

**关键元素:**
- **敌军情报:** 显示敌人类型和数量,给出推荐配置
- **编队配置:**
  - 显示每种无人机的数量
  - [+增加] [-减少]按钮调整数量
  - 显示当前编队数/上限(受指挥科技限制)
  - 显示库存可用数量
- **资源检查:**
  - 自动计算战斗所需弹药和电力
  - 显示当前库存/预计消耗
  - 充足✓/不足✗标识
- **昼夜提示:**
  - 显示当前时间
  - 预计战斗时长
  - 如果可能跨越昼夜,给出警告
- **战斗预测:**
  - 预计回合数
  - 胜率估算

**操作行为:**
- 调整编队数量(不能超过上限和库存)
- 点击[确认进攻] → 跳转到战斗界面
- 点击[返回地图] → 返回地图界面

## 四、通用UI元素

### 4.1 资源显示规则
- **实时更新:** 被动刷新,资源变动时触发更新
- **颜色标识:**
  - 充足: 白色/绿色
  - 不足: 黄色
  - 严重不足: 红色
- **格式:** `当前值/上限` 或 `当前值` (无上限物品)

### 4.2 状态指示器
- **✓** - 满足条件/已完成/正常
- **✗** - 不满足条件/未完成/失败
- **⚠️** - 警告
- **⏰** - 时间相关
- **⚡** - 电力相关

### 4.3 按钮状态
- **可用** - 绿色/正常颜色,可点击
- **不可用** - 灰色,禁用状态
- **警告** - 黄色,可点击但有风险
- **危险** - 红色,不可逆操作

### 4.4 交互反馈
- **点击按钮:** 无二次确认,直接执行
- **危险操作:** 拆除、删除资源等显示确认提示
- **加载/处理:** 显示简单的"处理中..."提示

### JavaScript切换逻辑
```javascript
function showScreen(screenName) {
  // 隐藏所有界面
  document.querySelectorAll('.screen').forEach(screen => {
    screen.style.display = 'none';
  });
  
  // 显示目标界面
  document.getElementById(screenName + '-screen').style.display = 'block';
  
  // 更新Tab栏激活状态
  updateTabBar(screenName);
}

function updateTabBar(activeTab) {
  // 更新Tab按钮的激活状态
  document.querySelectorAll('#tab-bar button').forEach(btn => {
    btn.classList.remove('active');
  });
  event.target.classList.add('active');
}
```

## 十、总结

本界面设计文档定义了《扩张前线》的完整UI结构:

**核心界面(5个Tab):**
1. 区域界面 - 主游戏界面,管理当前区域
2. 建造界面 - 建造新建筑
3. 科技树界面 - 研究科技
4. 仓库界面 - 管理资源上限
5. 地图界面 - 区域总览和战斗入口

**战斗相关界面(3个):**
6. 战斗准备界面
7. 战斗界面
8. 战斗结算界面

**设计原则:**
- 单页应用,无刷新切换
- 信息清晰,操作简单
- 无二次确认(除危险操作)
- 被动资源更新
- 文字+简单UI,专注策略

**技术实现:**
- HTML + CSS + JavaScript
- 分离文件结构
- 简单的显示/隐藏切换
- LocalStorage存档
